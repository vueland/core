# CApp

`CApp` — корневой layout-компонент библиотеки **core-ui**.

Используется как верхнеуровневый контейнер приложения и предоставляет внутренний API через `provide/inject`, включая
управление scroll (позиция, блокировка и восстановление).

---

## Базовое использование

```html

<c-app>
  <router-view/>
</c-app>
```

---

## Назначение компонента

`c-app` используется как **корневая обертка приложения** и отвечает за:

* формирование базового layout-контейнера
* централизованное управление scroll
* блокировку прокрутки (например, при открытии модалок)
* восстановление scroll-позиции без "прыжков"
* предоставление API дочерним компонентам

---

## Пример структуры

```html

<c-app>
  <c-toolbar/>
  <c-main>
    <router-view/>
  </c-main>
```

## Slots

| Slot    | Описание                       |
|---------|--------------------------------|
| default | Основное содержимое приложения |

```html

<c-app>
  <div>Контент приложения</div>
</c-app>
```

---

## Provide API

Компонент предоставляет объект по ключу `$APP_API_KEY`.

### Доступные методы

| Метод           | Тип            | Описание                                       |
|-----------------|----------------|------------------------------------------------|
| `getScrollTop`  | `() => number` | Возвращает текущую вертикальную позицию scroll |
| `getScrollLeft` | `() => number` | Возвращает текущую горизонтальную позицию      |
| `blockScroll`   | `() => void`   | Блокирует прокрутку страницы                   |
| `unblockScroll` | `() => void`   | Восстанавливает прокрутку и позицию            |

## Props

У компонента нет входных props.

---

## Events

Компонент не генерирует пользовательских событий.

---

## Особенности

* использует `position: fixed` для блокировки scroll без прыжков
* сохраняет текущую позицию scroll перед блокировкой
* восстанавливает scroll после разблокировки
* работает через `requestAnimationFrame` для корректного timing
* не зависит от того, где находится scroll (window)

---

## Рекомендации по использованию

* размещайте `c-app` в корне приложения
* используйте один `c-app` на приложение
* оборачивайте в него layout и страницы
* используйте `inject($APP_API_KEY)` в composables при необходимости

---

## Пример с inject

```ts
import { inject } from 'vue'
import { $APP_API_KEY } from '../../constants'

const app = inject($APP_API_KEY)
```
